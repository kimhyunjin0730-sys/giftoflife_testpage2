⚙️ 설정 가이드

Firebase Auth + Supabase DB 연결 방법을 단계별로 안내합니다.

1

Firebase 프로젝트 생성

console.firebase.google.com 접속 → "프로젝트 추가" 클릭 → 프로젝트 이름 입력 (예: goli-korea) → 계속 → Firebase 생성

2

Authentication 활성화

왼쪽 메뉴 "빌드 → Authentication" → "시작하기" → "이메일/비밀번호" 활성화 → "Google"도 활성화 (소셜 로그인용)

💡 Google 로그인 시 지원 이메일 주소를 입력해야 합니다.
3

앱 등록 및 Config 값 복사

프로젝트 개요 → "웹 앱 추가 ( 아이콘)" → 앱 이름 입력 → 등록 → 아래 코드 블록이 표시됩니다:

// 이런 형태로 표시됩니다: const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "your-project.firebaseapp.com", projectId: "your-project-id", storageBucket: "your-project.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };

이 값을 복사해두세요. "코드 교체" 탭에서 붙여넣기 위치를 안내합니다.

4

허가된 도메인 추가

Authentication → "Settings" → "Authorized domains" → "도메인 추가" → kimhyunjin0730-sys.github.io 입력 후 추가

✅ 이 작업을 해야 github.io 에서 Google 로그인이 동작합니다.
✅ 연결하면 자동으로 저장되는 테이블: donations (후원신청) · members (회원) · news_cache (뉴스 캐시)
1

Supabase 계정 만들기

supabase.com 접속 → 우측 상단 "Start your project" 클릭

→ GitHub로 로그인 (GitHub 계정 없으면 먼저 가입)

💡 GitHub 계정이 있으면 별도 가입 없이 바로 연동됩니다.
2

새 프로젝트 생성

로그인 후 대시보드에서 "New project" 클릭

Organization: 본인 계정 선택 Name: goli-korea Database Password: 강력한 비밀번호 입력 (기억해 두세요) Region: Northeast Asia (Seoul) ← 한국 서버로 선택

"Create new project" 클릭 → 약 1~2분 대기 (초록 체크가 뜨면 완료)

3

테이블 생성 — SQL Editor에서 실행

왼쪽 메뉴 "SQL Editor" 클릭 → 상단 "New query" 클릭 → 아래 SQL 전체 복사 후 붙여넣기 → 우측 상단 "Run" (또는 Ctrl+Enter)

-- ① 후원 신청 테이블 CREATE TABLE IF NOT EXISTS donations ( id BIGSERIAL PRIMARY KEY, name TEXT NOT NULL, email TEXT NOT NULL, phone TEXT, grade TEXT, amount TEXT, pay_type TEXT, depositor TEXT, pay_date DATE, pay_method TEXT, status TEXT DEFAULT 'pending', lang TEXT DEFAULT 'ko', created_at TIMESTAMPTZ DEFAULT NOW() ); -- ② 회원 테이블 CREATE TABLE IF NOT EXISTS members ( id BIGSERIAL PRIMARY KEY, firebase_uid TEXT UNIQUE, name TEXT, email TEXT UNIQUE NOT NULL, phone TEXT, grade TEXT DEFAULT 'supporter', matched_child TEXT, total_paid INTEGER DEFAULT 0, created_at TIMESTAMPTZ DEFAULT NOW() ); -- ③ 뉴스 캐시 테이블 (스크래핑 결과 24시간 저장) CREATE TABLE IF NOT EXISTS news_cache ( id BIGSERIAL PRIMARY KEY, lang TEXT NOT NULL, limit_count INTEGER NOT NULL, articles JSONB NOT NULL, source_url TEXT DEFAULT 'giftoflifeinternational.org', fetched_at TIMESTAMPTZ DEFAULT NOW(), UNIQUE(lang, limit_count) ); -- ④ RLS(Row Level Security) 활성화 ALTER TABLE donations ENABLE ROW LEVEL SECURITY; ALTER TABLE members ENABLE ROW LEVEL SECURITY; ALTER TABLE news_cache ENABLE ROW LEVEL SECURITY; -- ⑤ 익명 사용자 접근 정책 (삽입 허용) CREATE POLICY "anon_insert_donations" ON donations FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "anon_insert_members" ON members FOR INSERT TO anon WITH CHECK (true); -- ⑥ 뉴스 캐시는 읽기 + 쓰기 모두 허용 CREATE POLICY "anon_all_news_cache" ON news_cache FOR ALL TO anon USING (true) WITH CHECK (true);
✅ "Success. No rows returned" 메시지가 뜨면 성공입니다.
4

API 키 복사

왼쪽 메뉴 "Project Settings" (톱니바퀴 아이콘) → "API" 탭 → 두 값을 복사:

① Project URL https://abcdefghijklmnop.supabase.co ← 이 형태 ② Project API Keys → anon public eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJpc3... ← 긴 문자열
⚠️ "service_role" 키는 절대 사용 금지 — "anon public" 키만 사용하세요.
5

테이블 데이터 확인 방법

후원 신청이 들어오면: 왼쪽 메뉴 "Table Editor" → donations 테이블 클릭

뉴스 캐시 확인: "Table Editor" → news_cache 테이블 → articles 컬럼에 JSON 배열로 저장됨

✅ 뉴스 새로고침 후 news_cache 테이블에 데이터가 쌓이면 정상입니다.
다음 24시간 동안은 DB에서 즉시 불러와 API 호출 횟수를 절약합니다.
1

index.html 파일에서 Firebase Config 교체

파일 상단 <script type="module"> 블록 안의 아래 부분을 찾아 교체:

// ❌ 이 부분을 const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT_ID.firebaseapp.com", ... }; // ✅ Firebase 콘솔에서 복사한 값으로 교체 const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX", authDomain: "goli-korea.firebaseapp.com", projectId: "goli-korea", storageBucket: "goli-korea.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdef123456" };
2

Supabase Config 교체

JavaScript 섹션 상단의 SUPABASE CONFIG 주석 아래를 찾아 교체:

// ❌ 이 부분을 const SUPABASE_URL = 'YOUR_SUPABASE_URL'; const SUPABASE_KEY = 'YOUR_SUPABASE_ANON_KEY'; // ✅ Supabase Settings → API 에서 복사한 값으로 교체 const SUPABASE_URL = 'https://xxxxxxxxxxxx.supabase.co'; const SUPABASE_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.xxxx...';
3

뉴스 캐시 만료 시간 조정 (선택)

JS 섹션 상단의 NEWS_CACHE_HOURS 값을 원하는 시간으로 변경:

// 기본값 24시간 — 더 자주 최신 기사를 원하면 줄이세요 const NEWS_CACHE_HOURS = 24; // 예: 6 → 6시간마다 새로 긁기
4

저장 후 배포

GitHub에 commit & push → GitHub Pages가 자동으로 배포됩니다. 배포 후 firebase.google.com → Authentication → Authorized domains에 kimhyunjin0730-sys.github.io가 추가되어 있는지 확인하세요.

✅ 두 값(Firebase + Supabase) 모두 교체하면 자동으로 실제 DB에 저장됩니다.
뉴스 페이지 첫 방문 → 스크래핑 → news_cache 저장 → 이후 24시간은 DB에서 즉시 로드!
